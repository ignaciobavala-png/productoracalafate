# Internacionalizacion (i18n)

Sistema bilingue Espanol/Ingles sin librerias externas. Implementado con un mapa de texto centralizado y un store Zustand.

## Uso basico

```typescript
import { useOnboardingStore } from "@/store/onboarding-store";
import { t } from "@/lib/onboarding-text";

function MiComponente() {
  const language = useOnboardingStore((s) => s.language);
  return <h1>{t("hero.title", language)}</h1>;
}
```

## API: `t(keyPath, lang)`

```typescript
export function t(path: string, lang: Language): string
```

- `path`: ruta anidada con puntos, ej. `"hero.title"`, `"program.agenda.item1.title"`, `"shared.next"`.
- `lang`: `"es"` o `"en"`.
- Retorna el string correspondiente. Si la key no existe, retorna el path como fallback.

## Archivo de texto: `src/lib/onboarding-text.ts`

Estructura:

```typescript
export const onboardingText = {
  hero: {
    title: { es: "...", en: "..." },
    subtitle: { es: "...", en: "..." },
  },
  program: {
    agenda: {
      item1: {
        title: { es: "...", en: "..." },
        desc: { es: "...", en: "..." },
      },
    },
  },
  shared: {
    next: { es: "Siguiente", en: "Next" },
    back: { es: "Volver", en: "Back" },
  },
};

export type Language = keyof typeof onboardingText.hero.title;
// "es" | "en"
```

## Estructura de keys (~280 claves)

| Seccion | Key raiz | Contenido |
|---|---|---|
| Hero | `hero.*` | location, title, subtitle, tagline, date |
| Manifiesto | `manifesto.*` | label, title, p1, p2, photo |
| Programa | `program.*` | label, title, description, day1-3 labels, agenda item1-7 (title + desc) |
| Tarifa | `pricing.*` | label, title, description, price, currency, includes/excludes headers, note, cta |
| Stepper | `stepper.*` | Labels de los 4 pasos |
| Paso 1 | `stepPersonal.*` | title, subtitle, labels de 14 campos + companion section |
| Paso 2 | `stepDocuments.*` | title, subtitle, labels de dieta (6 opciones), dropzones, bio |
| Paso 3 | `stepPayment.*` | title, subtitle, invoice checkbox, payment method labels, terms checkbox + texto legal |
| Paso 4 | `stepConfirm.*` | title, subtitle, bloques de review, boton submit |
| Terminos | `terms.*` | 4 parrafos + politica de cancelacion |
| Exito | `success.*` | title, subtitle, message, home link |
| Footer | `footer.*` | description, contact, cta, design credit |
| Invitacion | `invitation.*` | title, subtitle, code/email labels, button, success |
| Compartido | `shared.*` | apply, back, next, submit, sending, photo |

## Como agregar texto nuevo

1. Agregar las claves ES/EN en `onboarding-text.ts` bajo la seccion correspondiente.

```typescript
// En onboardingText:
nuevaSeccion: {
  titulo: { es: "Hola", en: "Hello" },
}
```

2. Usar en un componente:

```typescript
t("nuevaSeccion.titulo", language)
```

No se requiere registro de tipos ni archivos adicionales. El tipado de `t()` acepta cualquier string como path.

## Idioma por defecto

`useOnboardingStore` inicializa con `language: "es"`. El usuario puede cambiar con `setLanguage("en")` via `LanguageSwitcher`.

## Convenciones

- Las keys usan camelCase y puntos como separador: `"stepPersonal.fullNameLabel"`.
- Los valores de restricciones alimenticias se almacenan en espanol en el store (`"Ninguna"`, `"Vegano"`), pero las labels en UI se traducen via `t()`.
- El texto legal (`terms.*`) esta completo en ambos idiomas.
